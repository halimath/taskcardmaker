Syntax reference

Start a new story

Every task must be part of a story. A task is always part of the story defined previously. To define a story add a line starting with S: followed by an identifier (a short label that is added to every task card) a pipe and a description. The description is optional; if you omit it the ID of the story is used (in such a case omit the pipe, too) 

S:ID-XXX|Test the taskcardmaker
S:ID-XXX

Add a task

Tasks are defined by writing text on a line on its own. If you want to add one or more tags to the bottom of the task, use a pipe to separate task text from tags. Tags should be separated by comma. 

Create some tasks
Create some more tasks|DESIGN
Print tasks|PDF,PRINT

Add a blocker task

Blocker tasks (shown in red instead of yellow) are defined just like normal tasks except that the line starts with B:. 

B:Create blocker tasks

Card Width

You can customize the card width. The card width is given in milli meters (mm). The default is 80 mm. Notice that too small or too large values will result in bad rendering and layout errors. Reasonable values are between 60 and 100 mm. 

# width 75

Font Size

You can customize the base font size that is used. The font size is given in points. The default is 14 pt. The given base font size will be used to calculate all actual font sizes (i.e. story identifiers have a larger font). 

# font_size 13

Enable/ Disable Checkboxes

You can enable/ disable checkboxes being printed to the lower right corner of every task card. The default is to add a checkbox. 

# checkboxes yes
# checkboxes no
